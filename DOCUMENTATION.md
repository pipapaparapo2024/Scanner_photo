# Документация приложения Scanner Photo

## Обзор
Scanner Photo — это мобильное приложение для сканирования документов с функцией распознавания текста (OCR), облачным хранением и историей сканирования. Приложение построено на современной архитектуре с использованием React Native для мобильного клиента и Node.js (Express) для серверной части.

## Архитектура проекта

### Структура репозитория
Проект организован как монорепозиторий с разделением на клиентскую и серверную части:

```
root/
├── mobile/                 # React Native приложение (клиент)
├── backend/                # Node.js Express приложение (сервер)
├── shared/                 # Общий код (типы, утилиты)
└── DOCUMENTATION.md        # Этот файл
```

### Мобильное приложение (`mobile/`)
Мобильный клиент разработан на React Native и следует методологии Feature-Sliced Design (FSD) для обеспечения масштабируемости.

**Структура папок:**
- `app/`: Инициализация, провайдеры контекста, навигация.
- `pages/`: Экранные компоненты (Страница входа, История сканов, Детали скана и др.).
- `widgets/`: Крупные самостоятельные UI-блоки.
- `features/`: Реализация бизнес-логики (камера, история, авторизация).
- `entities/`: Бизнес-сущности (User, ScanDoc).
- `shared/`: Переиспользуемые компоненты и утилиты.

**Ключевые технологии:**
- **React Native**: Основной фреймворк.
- **React Navigation**: Навигация (стек и табы).
- **Firebase Auth**: Аутентификация пользователей.
- **ML Kit / Tesseract**: Распознавание текста (OCR).

### Серверная часть (`backend/`)
Backend обеспечивает API для хранения данных сканирования, обработки изображений и синхронизации.

**Структура папок:**
- `src/index.ts`: Точка входа.
- `src/routes/`: Определение API маршрутов (scans, auth).
- `src/controllers/`: Логика обработки запросов.
- `src/services/`: Бизнес-логика (взаимодействие с БД, внешними API).
- `src/models/`: Схемы данных (Firestore/MongoDB).

**Ключевые технологии:**
- **Node.js & Express**: Серверная платформа.
- **Firestore**: База данных для хранения метаданных сканов.
- **Cloud Storage**: Хранение изображений.

## Функциональность

### Основные возможности
1.  **Сканирование документов**: Использование камеры устройства для получения изображений.
2.  **Распознавание текста (OCR)**: Извлечение текста из изображений.
3.  **История сканирования**: Просмотр ранее отсканированных документов.
4.  **Избранное**: Возможность отмечать важные сканы.
5.  **Массовое удаление**: Выбор нескольких элементов в истории для удаления.
6.  **Аутентификация**: Вход и регистрация через Email (Firebase).
7.  **Профиль пользователя**: Управление аккаунтом и настройки.

### Навигация
- После авторизации пользователь попадает на экран Аккаунта.
- Основная навигация осуществляется через нижние табы (История, Сканирование, Профиль).

## Инструкция по запуску (Setup & Run)

### Предварительные требования
- Node.js (v18 или выше)
- npm или yarn
- Android Studio (для запуска эмулятора Android)
- JDK 11 или 17

### 1. Установка зависимостей

**Backend:**
```bash
cd backend
npm install
```

**Mobile:**
```bash
cd mobile
npm install
```

### 2. Запуск Backend сервера
Сервер по умолчанию запускается на порту 4000.

```bash
cd backend
npm run dev
```
*Убедитесь, что переменные окружения (.env) настроены корректно (Firebase credentials и т.д.).*

### 3. Запуск Мобильного приложения

**Запуск Metro Bundler:**
```bash
cd mobile
npm start
```

**Запуск на Android:**
Откройте новое окно терминала:
```bash
cd mobile
npm run android
```

### Решение частых проблем
- **Ошибка Metro**: Если возникает ошибка разрешения модулей, попробуйте очистить кэш: `npm start -- --reset-cache`.
- **Ошибка Gradle**: Убедитесь, что в `android/local.properties` указан правильный путь к Android SDK.
