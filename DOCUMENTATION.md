## 1. Введение
Scanner Photo — это инновационное мобильное приложение, разработанное для мгновенного сканирования и распознавания текста (OCR) с использованием камеры смартфона. В эпоху цифровизации, когда скорость обработки информации имеет решающее значение, Scanner Photo предлагает пользователям простой и эффективный способ перевода бумажных документов в цифровой формат.

Приложение ориентировано на широкий круг пользователей: от студентов, конспектирующих лекции, до бизнес-профессионалов, обрабатывающих договоры и счета. Основная миссия проекта — сэкономить время пользователей, избавив их от ручного перепечатывания текста, и обеспечить надежное хранение цифровых копий.

## 2. Функциональность приложения
Приложение обладает богатым набором функций, обеспечивающих полный цикл работы с документами:

### 2.1. Сканирование и Распознавание (OCR)
- **Умная камера**: Автоматическое определение границ документа и коррекция перспективы (в разработке).
- **Высокая точность**: Использование передовых алгоритмов OCR (Google Cloud Vision API) для распознавания текста на различных языках, включая рукописный ввод.
- **Поддержка форматов**: Распознавание печатного текста, таблиц и сложных макетов.

### 2.2. Управление Историей
- **Локальное хранилище**: Мгновенный доступ к последним сканам даже без интернета благодаря кэшированию.
- **Синхронизация**: Автоматическая синхронизация истории сканирований между устройствами через облачную базу данных.
- **Поиск и Фильтрация**: Удобный поиск по содержимому текста и дате сканирования.
- **Избранное**: Возможность отмечать важные документы для быстрого доступа.

### 2.3. Работа с Результатами
- **Редактирование**: Возможность скорректировать распознанный текст вручную.
- **Экспорт**:
    - Копирование текста в буфер обмена.
    - Экспорт в PDF для печати или отправки.
    - Экспорт в TXT для дальнейшего редактирования.
- **Поделиться**: Отправка результатов через мессенджеры, почту или соцсети.

### 2.4. Пользовательский Опыт
- **Темная и Светлая темы**: Автоматическое переключение в зависимости от настроек устройства.
- **Многоязычность**: Полная локализация интерфейса (Русский, Английский).
- **Безопасность**: Авторизация через Email/Firebase для защиты данных пользователя.

## 3. Архитектура проекта
Проект спроектирован с учетом требований к масштабируемости и поддерживаемости, следуя принципам Feature-Sliced Design (FSD) или модульной архитектуры.

### Структура Мобильного Приложения (Mobile Monorepo)
- **app/**: Инициализация приложения, навигация, глобальные провайдеры (темы, авторизация).
- **pages/**: Компоненты страниц, собирающие функционал из виджетов и фич.
- **widgets/**: Самостоятельные блоки UI (например, списки сканов, формы).
- **features/**: Бизнес-логика и интерактивные элементы (например, `scan-history`, `auth-flow`).
- **entities/**: Бизнес-сущности (модели данных `ScanDoc`, API запросы).
- **shared/**: Переиспользуемый код (UI-кит, утилиты, конфиги, API клиенты).

### Взаимодействие с Бэкендом
- **REST API**: Обмен данными происходит через защищенный REST API.
- **Аутентификация**: Использование JWT токенов, выдаваемых Firebase, для валидации запросов на сервере.
- **База Данных**: Хранение метаданных сканов и ссылок на изображения в NoSQL базе данных (Firestore/MongoDB) для обеспечения высокой скорости чтения и гибкости схемы.

## 4. Технологический стек

### Мобильное Приложение (Frontend)
- **Платформа**: React Native (Expo SDK 50+) — кроссплатформенная разработка для iOS и Android.
- **Язык**: TypeScript — строгая типизация для повышения надежности кода.
- **Навигация**: React Navigation (Native Stack + Bottom Tabs).
- **Анимации**: React Native Reanimated 3.
- **Стейт-менеджмент**: React Hooks & Context API.
- **Сетевой слой**: Axios для HTTP запросов.
- **Хранение данных**: AsyncStorage для кэширования + Firebase SDK.

### Бэкенд и Инфраструктура
- **Сервер**: Node.js (NestJS или Express) — производительный и масштабируемый сервер.
- **База данных**: Google Firestore или MongoDB.
- **OCR Engine**: Google Cloud Vision API.
- **Auth**: Firebase Authentication.
- **CI/CD**: GitHub Actions (планируется).

## 5. Инструкция по запуску
Для развертывания проекта локально выполните следующие шаги:

### Предварительные требования
1. **Node.js**: Версия 18.x или выше.
2. **Пакетный менеджер**: npm или yarn.
3. **Мобильная среда**:
   - Android Studio (для Android эмулятора).
   - Xcode (для iOS симулятора, только macOS).
   - Expo Go (для запуска на реальном устройстве без сборки).

### Установка
1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd scanImg
   ```
2. Перейдите в папку мобильного приложения (если это монорепозиторий):
   ```bash
   cd mobile
   ```
3. Установите зависимости:
   ```bash
   npm install
   ```

### Запуск
1. Запустите Metro Bundler:
   ```bash
   npm start
   ```
2. Выберите платформу:
   - Нажмите `a` для запуска Android эмулятора.
   - Нажмите `i` для запуска iOS симулятора.
   - Отсканируйте QR-код через приложение Expo Go на телефоне.

### Сборка (Build)
Для создания APK/AAB файлов:
```bash
npm run android:build
```
